Form Input Rules

The input rules directive allows you to add additional validation rules to the standard inputs.

Put 'input-rules="config"' as an attribute on the input that you want to validate and pass a configuration object.

This directive can only be used in combination with the ng-model directive, inside a form or fieldset.

Configuration object

Key Type Description
excluded array This is an array of explicitly excluded strings. If the input matches one of thos, it will become invalid.
patterns array This is a list of JavaScript RegEx. If the input value does not pass all expressions, the input will become invalid.

Example

HTML

<bk-fieldset ng-form="forms.example">
    <bk-form-group>
        <bk-form-item>
            <bk-input type="text" name="example" ng-model="inputModel" ng-required="true" input-rules="inputRulesConfig" state="{{ forms.example['example'].$valid ? '' : 'error' }}" placeholder="Type something"></bk-input>
        </bk-form-item>
    </bk-form-group>
</bk-fieldset>

JavaScript

$scope.forms = { example: {} }; $scope.inputs = { inputModel: '' }; $scope.inputRulesConfig = { excluded: ['forbidden', 'words'], patterns: ['^[^/]*$'], // Does not allow '/' };
