Shellbar

The shellbar includes support for branding, product navigation, search, notifications, and user settings. This is a composite component comprised of mandatory and optional elements. The Shellbar should always be placed inside the shell layout container and there must always be only one shellbar for the entire application.

Attributes - Shellbar

Name Type Required Description
size string False Set the horizontal paddings. Possible options are 's', 'm', 'l', 'xl' and 'responsive'.

Attributes - Shellbar Group

Name Type Required Description
position string False Position of the group. Possible options are 'left', 'center' and 'right'.
shrink boolean False The group will shrink to it's minimum content size if true. If false, it will expand to it's maximum size.

Attributes - Shellbar Action

Name Type Required Description
grow boolean False If the action should grow horizontally.
shrink boolean False If the action should shrink horizontally.
mobile boolean False For actions only, visible on small shellbar (--s).
desktop boolean False For actions only, visible on desktop screens (--m/l/xl).

Examples

BlimpKit
$scope.isSearchFocused = false; $scope.focusSearch = () => { $scope.isSearchFocused = true; }; $scope.blurSearch = () => { $scope.isSearchFocused = false; };
